What is React JS?
    • React JS is an open-source JavaScript library developed and maintained by Facebook.
    • It’s widely used for building user interfaces, especially for single-page applications and complex web interfaces.
    • It breaks down a whole website into small, manageable pieces called components.
    • It enables developers to create dynamic, responsive, and reusable UI components, enhancing the efficiency of the development process.
Why React JS?
    • Speed: React uses the Virtual DOM, allowing efficient updates and rendering, resulting in faster and more responsive user interfaces.
    • Reusability: React follows a component-based approach where components can be reused. This leads to cleaner code, easier maintenance, and better collaboration among team members.
    • Job Market: React has high demand. Companies like Netflix, Airbnb, Instagram, etc. use React.
    • Rich Ecosystem and Tooling: React is supported by a vast ecosystem of libraries, tools, and frameworks that enhance its capabilities and help developers build robust applications more efficiently.

What is DOM?
    - DOM(Document Object Model) is a tree like structure that represents your HTML elements in the browser
    - This becomes a DOM tree like
    - Changing or updating this structure directly (called real DOM manipulation) is slow especially for big applications

 What is the Virtual DOM?
    - The Virtual DOM is a copy of the real DOM kept in memory by react.
    - It's a JavaScript object that mirrors the structure of the real DOM. But doesn't touch the screen immediately.

React never updates the DOM directly. Instead it
    - Creates a virtual version of the Ul
    - Updates that virtual version when state or props change
    - Compares it with the previous version
    - Finds what changed
    - Efficiently updates only those parts in the real DOM.

When is the virtual DOM used?
    •⁠  ⁠Every time something changes in your app (like a button click or input text change) React.
    +------------------+        +------------------+        +------------------+
    | User Interaction | -----> |   Virtual DOM    | -----> |   Original DOM   |
    |------------------|        |------------------|        |------------------|
    |      Client      |        | React compares   |        | Real webpage     |
    | User clicks/type |        | changes here     |        | gets updated     |
    +------------------+        +------------------+        +------------------+

    Flow:
    User Action --> Virtual DOM --> Original DOM Update
    •⁠  ⁠Re-renders the component to create a new Virtual DOM
    •⁠  ⁠React compares the new virtual DOM with the old virtual DOM to find what changed
    •⁠  ⁠Calculates the minimum number of changes required
    •⁠  ⁠Applies those changes to the real DOM

Why is this important?
Updating the real DOM is slow because
    - It involves recalculating styles, layouts and reflows
    - The entire page may be re-rendered unnecessarily

React's virtual DOM makes updates faster and smoother by
    - Avoiding unnecessary work
    - Making Dom updates batched and optimized

    •⁠  ⁠Instead of a manipulating the browser's Dom directly, React creates a virtual Dom in memory. Where it does all the necessary manipulating before making the changes in the Browser dom.

    and updates only the changed part in Real DOM.
    -The reconciliation process in React is the mechanism by which React determines what changes need to be made to the Real DOM in response to updates in the Virtual DOM.
    -It involves comparing the current Virtual DOM representation of a component with its previous representation to identify the differences and efficiently update the Real DOM.
Here's a step-by-step overview of the reconciliation process in React:
    Component Render:
        • When a component's state or props change, React re-renders the component and its children to generate a new Virtual DOM representation.
    Diffing Algorithm:
        • React performs a “diffing” or comparison process between the new Virtual DOM tree and the previous Virtual DOM tree for the component.
        • It recursively traverses the component tree and compares each node (element) in the new tree with its counterpart in the previous tree.
Identifying Changes:
    React identifies three types of changes:
        • Added Nodes
        • Removed Nodes
        • Updated Nodes
    Applying Changes:
        • React efficiently applies the identified changes to the Real DOM, updating only the affected parts without re-rendering the entire DOM tree.
        • This process involves adding, removing, or updating DOM elements and attributes as necessary to reflect the changes in the UI.

By performing this reconciliation process, React ensures that the UI remains in sync with the application state while minimizing unnecessary updates to the Real DOM, leading to better performance and a smoother user experience.

Difference b/w Real DOM and Virtual DOM

    | Real DOM                                           | Virtual DOM                                                          |
    | -------------------------------------------------- | -------------------------------------------------------------------- |
    | Represents the actual structure of a web page.     | A lightweight, in-memory representation.                             |
    | Created by the browser when a page loads.          | Created and maintained by React for each component.                  |
    | Inefficient for frequent updates.                  | Efficient for frequent updates.                                      |
    | Updating is slow and resource-intensive.           | Updating is fast and less resource-intensive.                        |
    | Directly manipulates the browser’s display.        | Indirectly manipulates the browser’s display through reconciliation. |
    | Requires re-rendering the entire tree for updates. | Requires minimal updates for changes.                                |
